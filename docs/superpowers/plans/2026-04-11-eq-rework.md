# EQ Window Rework — 2026-04-11

**Goal:** Restyle the unskinned equalizer window to look closer to classic Winamp: taller response curve with sharper bends, paler centerline, thinner curve/line, reworked preamp column (right-shifted, full "PREAMP" label, orange dB markers, decorative stripes), and frequency slider restyle (thinner track, green→red gradient, pale gridlines between columns, brighter labels, lowered labels).

**Architecture:** All changes live in `EqualizerView.swift`, `EQResponseView.swift`, `WinampSlider.swift` (track drawing), and possibly `WinampTheme.swift` (new colors). No audio-engine changes.

**Branch:** `feature/eq-rework` off `main` after merging `feature/standard-ui-tweaks-round2`.

**Process:** One item → `xcodebuild` → commit with a one-line message. The user will request screenshots if needed — do not prompt for them.

---

## Recommended order

Mechanical → cosmetic → risky:

1. **Item A — Revert EQ range to ±12 dB** (and reclamp presets). Foundation for everything else.
2. **Item B — Response curve: taller, sharper, paler centerline, thicker curve.**
3. **Item C — Preamp column: right-shift, "PREAMP" caps, ±12 db orange labels, decorative stripes.**
4. **Item D — Frequency slider track: thinner, green→red gradient.**
5. **Item E — Band gridlines: pale horizontal ticks at top/mid/bottom between columns.**
6. **Item F — Labels polish: brighter band labels, lower both band labels and "PREAMP" so max-down sliders don't cover them.**

## Item A — Revert EQ range to ±12 dB

**What:** Round 1 bumped the slider range to ±20 dB and re-derived 17 presets against that range. User now wants the shipped classic Winamp range of ±12 dB, with all presets clamped accordingly.

**Where:**
- `Wamp/UI/EqualizerView.swift` — wherever slider `minValue`/`maxValue` are set (grep `20` / `-20` / `minValue` / `maxValue`).
- `Wamp/Audio/AudioEngine.swift` — if EQ band gain is clamped there.
- Preset definitions: 17 classic presets (from round 1 commit `c360d61`). Reclamp each band value to `[-12, 12]`.

**Change:**
- Band slider min/max: `-12 ... +12`.
- Preamp slider min/max: `-12 ... +12`.
- Preset bands: if a value falls outside ±12, clamp to ±12. (Classic Winamp presets are natively ±12 anyway — likely this is just reverting whatever scaling was added.)
- Update dB labels on the preamp column to `+12 db`, `0 db`, `-12 db` (done in Item C, but values need to match here).

**Verify:** Each preset still loads and plays without clipping; curve view reflects ±12 max range.

## Item B — Response curve: taller, sharper, paler centerline, thicker curve

**What:**
- Grow `EQResponseView` height by ~25% (round-2 set it to 19; new target ~24).
- Curve bends closer to the top/bottom edges at max/min gain — reduce the vertical margin.
- Centerline paler (`white α 0.35` instead of fully opaque).
- Centerline stays at 1 px, curve renders at 1.4 px so the curve visually dominates the line.

**Where:**
- `Wamp/UI/EqualizerView.swift` — `responseView.frame = NSRect(..., height: 19)` → height 24. Verify no clipping against sliders below or title bar above (fallback heights 22 / 20 if it clips).
- `Wamp/UI/Components/EQResponseView.swift` — `drawBuiltIn()`:
  - Keep the 10 vertical pale gridlines (`white α 0.18`).
  - Centerline `fill()` alpha `0.35` (was `1.0`).
  - Curve: `path.lineWidth = 1.4` (was `1.2`); change vertical margin from `height/2 - 2` to `height/2 - 1` (or `- 0.5`) so ±12 dB hits near the actual edges.
  - Normalization: with Item A range now ±12, use `gain / 12` instead of `gain / 20`.

## Item C — Preamp column: right-shift, "PREAMP" caps, ±12 db orange labels, decorative stripes

**What:**
- Preamp slider moves to the right of where it is today (give it some breathing room from the left edge).
- The vertical "PREAMP" label becomes `PREAMP` caps, full word (currently it may be clipped or abbreviated; verify).
- The three dB markers to the right of the preamp slider become `+12 db`, `0 db`, `-12 db`, rendered in a slightly orange tint (e.g. `NSColor(hex: 0xD89060)` or similar — orange-gold).
- Decorative light horizontal hairlines flanking the preamp column, matching the classic skin eqmain.bmp pattern (short 1 px pale lines at the top and bottom of the preamp slider area).

**Where:**
- `Wamp/UI/EqualizerView.swift` — `preampSlider.frame`, label frames (`viewWithTag(210)` etc.), and a new `drawBuiltIn()`-style helper on EqualizerView (or a subview) to draw the hairlines.
- Labels: whichever text fields render `+20 / 0 / -20` — change values and color.
- `Wamp/UI/WinampTheme.swift` — new `preampLabelOrange` color constant.

## Item D — Frequency slider track: thinner, green→red gradient

**What:**
- The track ("колба") on each frequency slider is ~20% thinner than today.
- Track background is a vertical gradient: green at the bottom → red at the top, as on classic Winamp eqmain.bmp.
- Applies to the unskinned (built-in) drawing path of `WinampSlider` only. Skinned path is unchanged.

**Where:**
- `Wamp/UI/Components/WinampSlider.swift` — the `drawBuiltIn()` (or equivalent) function that paints the slider track. Find the rect that represents the track, shrink its width by 20%, and replace its fill with `NSGradient(starting: greenBottom, ending: redTop)` drawn at 90°.
- Ensure the change only kicks in for the vertical sliders used by the EQ (add a flag if WinampSlider is shared with volume/balance, which use horizontal tracks — likely already branched by orientation).
- Possibly add theme colors `eqTrackBottom` (green) and `eqTrackTop` (red).

---

## Item E — Band gridlines: pale horizontal ticks at top/mid/bottom between columns

**What:** Classic Winamp eqmain.bmp has three faint horizontal dashes running across the entire band area at the top, the vertical center, and the bottom — they appear as tick marks between adjacent band columns. Render those in the unskinned EQ view.

**Where:**
- `Wamp/UI/EqualizerView.swift` — new `override func draw(_:)` guarded by `!WinampTheme.skinIsActive`, drawing three horizontal pale 1 px lines across the band slider area at top, mid, and bottom of the slider track rect. Use a very pale color (e.g. `white α 0.12`).

**Notes:**
- These are *behind* the sliders (draw first, then the sliders sit on top).
- `draw(_:)` on a container NSView runs before subviews are composited, so that ordering works naturally.
